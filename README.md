## <img src="https://img.icons8.com/fluency/32/idea.png" width="26"/> What is LEGIONHERCULES?

<img src="https://media.giphy.com/media/3oKIPnAiaMCws8nOsE/giphy.gif" width="380" align="right" alt="hacking gif"/>

You know how Claude Code and OpenCode are genuinely impressive, but cost money and phone home to someone else's server? LEGIONHERCULES is the version that runs on your machine, uses models you pull yourself, and costs nothing after setup.

It's a CLI framework. Agents run autonomously — they don't just respond to prompts, they plan and execute multi-step tasks. Multiple agents can run in parallel, so a coder agent and a research agent can work at the same time instead of queuing up.

The terminal UI is built with Rich, so it doesn't look like a debug log. The CLI is Typer. Ollama handles the inference locally.

That's basically the whole thing. No magic words. No "ecosystem." Just agents, tools, and your hardware.

## <img src="https://img.icons8.com/fluency/32/box.png" width="26"/> Installation

<div align="center">
<img src="https://media.giphy.com/media/26tn33aiTi1jkl6H6/giphy.gif" width="420" alt="install gif"/>
</div>

<br/>

### Prerequisites

Before anything else, make sure you have:

- **Python 3.9+** — check with `python --version`
- **[Ollama](https://ollama.com)** installed and running on your machine

> No Ollama yet? Install it from [ollama.com](https://ollama.com). It takes about two minutes.

<br/>

### <img src="https://img.icons8.com/fluency/22/download.png"/> From PyPI

```bash
pip install legionhercules
```

That's it. Shortest path to running.

<br/>

### <img src="https://img.icons8.com/fluency/22/source-code.png"/> From Source

If you want to poke at the internals or contribute:

```bash
git clone https://github.com/deathlegion/legionhercules.git
cd legionhercules
pip install -e ".[dev]"
```

## <img src="https://img.icons8.com/fluency/32/rocket.png" width="26"/> Quick Start

<div align="center">

<img src="https://media.giphy.com/media/077i6AULCXc0FKTj9s/giphy.gif" width="460" alt="quick start terminal gif"/>

<br/>

<img src="https://readme-typing-svg.demolab.com?font=Fira+Code&size=15&pause=900&color=34D399&center=true&vCenter=true&width=480&lines=Step+1%3A+Start+Ollama;Step+2%3A+Pull+a+model;Step+3%3A+legionhercules+chat" alt="Quick Start Typing" />

</div>

<br/>

**Step 1 — Start Ollama**

```bash
ollama serve
```

**Step 2 — Pull a model** (only needed once)

```bash
ollama pull llama3.2
```

**Step 3 — Start a session**

```bash
# Interactive session
legionhercules chat

# Or fire off a single message and exit
legionhercules chat "What can you do?"
```

## <img src="https://img.icons8.com/fluency/32/book.png" width="26"/> Usage

<div align="center">
<img src="https://media.giphy.com/media/dWesBcTLavkZuG35MI/giphy.gif" width="440" alt="coding examples gif"/>
</div>

<br/>

### Switching Models

```bash
legionhercules chat --model codellama
legionhercules chat --model mistral
legionhercules chat --model llama3.1
```

Pull whatever model you want from Ollama's library. The `--model` flag accepts any name that `ollama list` shows.

<br/>

### Switching Agents

```bash
# Coder agent — reads files, runs bash, edits code
legionhercules chat --agent coder

# Researcher agent — searches the web and summarizes
legionhercules chat --agent researcher
```

Agents are different system prompts plus tool configurations. The coder gets file/bash access. The researcher gets web search.

<br/>

### Managing Models

```bash
# See what you have locally
legionhercules models --list

# Pull a new one
legionhercules models --pull llama3.1
```

## <img src="https://img.icons8.com/fluency/32/blueprint.png" width="26"/> Architecture

<div align="center">
<img src="https://media.giphy.com/media/xT9IgzoKnwFNmISR8I/giphy.gif" width="380" alt="architecture gif"/>
</div>

<br/>

Five layers, each doing one thing:

```
╔═════════════════════════════════════════════╗
║               CLI  Layer                    ║
║          Rich Terminal Interface            ║
╠═════════════════════════════════════════════╣
║            Orchestrator                     ║
║      Parallel Task + Agent Manager          ║
╠═════════════════════════════════════════════╣
║               Agents                        ║
║     Reasoning  ·  Tool Calls  ·  LLM        ║
╠═════════════════════════════════════════════╣
║               Tools                         ║
║      File  ·  Bash  ·  Web Search           ║
╠═════════════════════════════════════════════╣
║           LLM Provider                      ║
║          Ollama  (local)                    ║
╚═════════════════════════════════════════════╝
```

The CLI talks to the Orchestrator. The Orchestrator spawns and manages Agents. Agents call Tools. Tools run locally or hit DuckDuckGo. The LLM never leaves your machine.

<div align="center">

| <img src="https://img.icons8.com/fluency/22/agent-profile.png"/> | Agent | Best for | Tools it uses |

## <img src="https://img.icons8.com/fluency/32/settings.png" width="26"/> Configuration

<div align="center">
<img src="https://media.giphy.com/media/QpVUMRUJGokfqXyfa1/giphy.gif" width="340" alt="config gif"/>
</div>

<br/>

LEGIONHERCULES picks up a config file from `~/.legionhercules/config.toml`. You don't need one to get started — defaults work fine — but here's what you can tune:

```toml
[llm]
provider = "ollama"
model = "llama3.2"
base_url = "http://localhost:11434"

[agents]
default = "coder"
max_parallel = 4

[tools]
bash_timeout = 30
web_search_results = 5
```

Override anything per-command with flags. The config just sets your defaults.

PRs are welcome. Issues too — especially bug reports with reproduction steps. Read the [Contributing Guide](https://legionhercules.dev/contributing) before you open a PR, mostly so we don't end up duplicating work.

### Dev Setup

```bash
git clone https://github.com/deathlegion/legionhercules.git
cd legionhercules

python -m venv venv
source venv/bin/activate     # Windows: venv\Scripts\activate

pip install -e ".[dev]"
pytest
```

Tests should all pass on a fresh clone. If they don't, open an issue.

### Ways to help

- Bug fixes (check existing issues first)
- New built-in tools
- Agent presets for specific workflows
- Documentation improvements
- Performance work on the parallel executor

## <img src="https://img.icons8.com/fluency/32/faq.png" width="26"/> FAQ

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Do I need a GPU?</b></summary>
<br/>
No. Ollama runs on CPU. It's slower without a GPU, but it works. A 7B parameter model on a modern laptop CPU is usable — not fast, but usable. If you have a GPU, Ollama will use it automatically.
<br/><br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Which model should I start with?</b></summary>
<br/>
<code>llama3.2</code> for general use. <code>codellama</code> if you're mostly doing code work. <code>mistral</code> if you want something a bit faster and lighter. All three are solid.
<br/><br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Is it actually free?</b></summary>
<br/>
Yes. The framework is MIT-licensed. Ollama is free. The models are free to pull and run. The web search uses DuckDuckGo's public API with no key required. The only cost is electricity.
<br/><br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Can I add my own tools?</b></summary>
<br/>
Yes. The tool interface is straightforward — check the <a href="https://legionhercules.dev/tools">Tools docs</a> for how to register a new tool and make it available to agents.
<br/><br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Does it work on Windows?</b></summary>
<br/>
Yes, with some caveats. The bash tool falls back to cmd/PowerShell on Windows. The rest works the same. Check the installation guide for Windows-specific setup steps.
<br/><br/>
</details>
