## <img src="https://img.icons8.com/fluency/32/idea.png" width="24"/> What is LEGIONHERCULES?

<img src="https://media.giphy.com/media/3oKIPnAiaMCws8nOsE/giphy.gif" width="340" align="right" alt="hacking"/>

You know those AI coding tools that are genuinely useful, but you're paying monthly and your code is going through someone else's server? LEGIONHERCULES is the version that doesn't do any of that.

It runs on your hardware. Uses Ollama for inference — completely local. No API key to generate, no billing page to visit, no usage cap to hit. You pull a model once, and it stays on your machine.

The agents aren't just chatbots with a tool call bolted on. They plan multi-step tasks, pick the right tools, run them, read the output, and keep going until the job is done. Multiple agents can run in parallel too — so a coder and a researcher can work at the same time without waiting on each other.

Terminal UI is Rich. CLI is Typer. Inference is Ollama. That's the full stack.

## <img src="https://img.icons8.com/fluency/32/rocket.png" width="24"/> Quick Start

<div align="center">

<img src="https://media.giphy.com/media/077i6AULCXc0FKTj9s/giphy.gif" width="480" alt="quick start terminal gif"/>

<img src="https://readme-typing-svg.demolab.com?font=Fira+Code&size=14&pause=900&color=C084FC&center=true&vCenter=true&width=500&lines=Three+steps.+Under+five+minutes.;No+accounts.+No+dashboards.+No+nonsense." alt="Quick Start Typing"/>

<br/>

**1 — Start Ollama**

```bash
ollama serve
```

Keep this running in the background. It's the inference server.

<br/>

**2 — Pull a model** *(once only)*

```bash
# General purpose — good starting point
ollama pull llama3.2

# Code-focused
ollama pull codellama

# Faster and lighter
ollama pull mistral
```

<br/>

**3 — Run it**

```bash
# Drop into an interactive session
legionhercules chat

# One-shot message
legionhercules chat "Refactor my main.py to use async/await"

# Pick a specific agent
legionhercules chat --agent coder "Fix the bug in utils.py"
```

## <img src="https://img.icons8.com/fluency/32/book.png" width="24"/> Usage

<div align="center">
<img src="https://media.giphy.com/media/dWesBcTLavkZuG35MI/giphy.gif" width="460" alt="usage gif"/>
</div>

<br/>

### Picking a Model

```bash
legionhercules chat --model codellama
legionhercules chat --model mistral
legionhercules chat --model llama3.1
legionhercules chat --model phi3
```

The `--model` flag takes anything `ollama list` shows. If the model isn't pulled yet, run `ollama pull <name>` first.

<br/>

### Picking an Agent

```bash
# Coder — file access, bash, code edits
legionhercules chat --agent coder

# Researcher — web search, summarization
legionhercules chat --agent researcher

# Default — everything
legionhercules chat --agent default
```

Agents aren't magic. They're different system prompts with different tool sets. The coder gets file + bash access. The researcher gets web search. The default agent gets all tools.

<br/>

### Model Management

```bash
# What's installed locally
legionhercules models --list

# Pull something new
legionhercules models --pull llama3.1

# Remove a model
ollama rm codellama
```

<br/>

### Combining Flags

```bash
# Researcher agent, mistral model, single task
legionhercules chat --agent researcher --model mistral "Summarize recent news on local LLMs"
```

## <img src="https://img.icons8.com/fluency/32/blueprint.png" width="24"/> Architecture

<div align="center">
<img src="https://media.giphy.com/media/xT9IgzoKnwFNmISR8I/giphy.gif" width="400" alt="architecture gif"/>
</div>

<br/>

Five layers. Each one does one thing and hands off to the next:

```
╔══════════════════════════════════════════════════╗
║                   CLI Layer                      ║
║           Rich Terminal  ·  Typer CLI            ║
╠══════════════════════════════════════════════════╣
║               Orchestrator                       ║
║       Parallel Task Manager · Agent Spawner      ║
╠══════════════════════════════════════════════════╣
║                  Agents                          ║
║      Reasoning  ·  Tool Selection  ·  LLM Loop  ║
╠══════════════════════════════════════════════════╣
║                  Tools                           ║
║       File  ·  Bash  ·  Web Search               ║
╠══════════════════════════════════════════════════╣
║              LLM Provider                        ║
║           Ollama  (100% local)                   ║
╚══════════════════════════════════════════════════╝
```

The CLI layer takes your input and passes it to the Orchestrator. The Orchestrator spawns agents and runs them in parallel if needed. Each Agent runs a loop: get task → pick tool → call it → read result → decide next step. Tools execute locally (file/bash) or hit DuckDuckGo. The LLM never talks to an external server.

## <img src="https://img.icons8.com/fluency/32/settings.png" width="24"/> Configuration

<div align="center">
<img src="https://media.giphy.com/media/QpVUMRUJGokfqXyfa1/giphy.gif" width="360" alt="config gif"/>
</div>

<br/>

Config lives at `~/.legionhercules/config.toml`. You don't need it to start — defaults cover the basics — but it's how you set permanent preferences so you're not passing flags every time.

```toml
[llm]
provider   = "ollama"
model      = "llama3.2"
base_url   = "http://localhost:11434"

[agents]
default     = "coder"
max_parallel = 4

[tools]
bash_timeout        = 30
web_search_results  = 5
file_read_max_lines = 500

[ui]
theme       = "dark"
show_tokens = true
```

Everything here can be overridden per-command with flags. The config just means you don't have to keep typing `--model codellama` every time.

## <img src="https://img.icons8.com/fluency/32/faq.png" width="24"/> FAQ

<div align="center">
<img src="https://media.giphy.com/media/3oEdva9BUHPHz2sBDO/giphy.gif" width="340" alt="faq gif"/>
</div>

<br/>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Do I need a GPU?</b></summary>
<br/>

No. Ollama runs fine on CPU. It's slower — a 7B model on a modern laptop CPU takes a few seconds per response — but it works. If you have a GPU, Ollama detects and uses it automatically. M-series Macs are particularly good here; Ollama uses Metal and the unified memory makes smaller models genuinely fast.

<br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Which model should I start with?</b></summary>
<br/>

`llama3.2` for general use — it's a solid default. `codellama` if most of your work is code. `mistral` if you want something faster and lighter. Don't overthink it; you can swap models anytime with `--model`.

<br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Is it actually free?</b></summary>
<br/>

Yes. MIT license. Ollama is free. Models are free to pull and run. `web_search` uses DuckDuckGo's public interface with no key required. The only running cost is electricity and disk space for the model weights.

<br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Can I write my own tools?</b></summary>
<br/>

Yes. The tool interface is a Python function with a docstring and a decorator. Check the [Tools docs](https://legionhercule.pages.dev/tools) for the exact pattern. Once registered, any agent can call your tool the same way it calls the built-ins.

<br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> Does it work on Windows?</b></summary>
<br/>

Yes. The `bash` tool falls back to cmd/PowerShell on Windows. A few things behave slightly differently (paths, shell syntax), but the rest works the same. See the [Installation Guide](https://legionhercule.pages.dev/installation) for Windows-specific setup steps.

<br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> How many agents can I run in parallel?</b></summary>
<br/>

That's `max_parallel` in your config. Default is 4. The real limit is your RAM and how fast your CPU (or GPU) can run inference. On a machine with 16GB RAM running a 7B model, 2-3 parallel agents is comfortable. 4+ is possible but you'll feel it.

<br/>
</details>

<details>
<summary><b><img src="https://img.icons8.com/fluency/18/question-mark.png"/> What if Ollama isn't running?</b></summary>
<br/>

You'll get a connection error immediately. Run `ollama serve` in a separate terminal first. You can also set Ollama to start automatically on login — check Ollama's docs for your platform.

<br/>
</details>

### Dev Setup

```bash
git clone https://github.com/deathlegionteamlk/legionhercules.git
cd legionhercules

python -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate

pip install -e ".[dev]"

```

Tests pass on a fresh clone. If they don't, open an issue with your Python version and OS.
